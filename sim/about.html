What is ViSiOMlab?

ViSiOMlab — a Virtual Simulator from the Ocular Motor lab at UC Berkeley — is a biophysically grounded computational model of how the brain controls eye movements. It simulates the full oculomotor pathway (semicircular canals, velocity storage, the neural integrator, saccades, smooth pursuit, vergence, and the eye plant) and renders the predicted eye movements as interactive plots and a 3-D avatar.

What you can do

  • Describe a scenario in plain English — a patient, a clinical test, or a head- and target-motion sequence — and see the predicted eye movements.
  • Model disorders by adjusting physiological parameters (e.g. vestibular, cerebellar, nerve/nucleus palsies) and watch how the eye movements change.
  • Compare two scenarios side by side.
  • Inspect the model's parameters and states & activations, and browse validation benchmarks and a gallery of examples.

How it works

The model is a differentiable state-space simulation built in JAX: each subsystem is a small dynamical model wired together in the signal-flow order of the real oculomotor system. A plain-English description is turned into a stimulus and parameter set, the dynamics are integrated, and the resulting eye, head, target, and scene signals are drawn as zoomable, cursor-synced plots alongside a head-fixed and world view of an avatar.

Simulation inputs

A scenario is built from four time-segment channels — head, target, and scene motion plus visual context — together with optional patient parameter changes and a plot selection. Each channel is a list of segments concatenated in time, and the trial lasts as long as the longest channel. Describing a scenario in plain English fills these in automatically, but every field below can be set directly (via the JSON a description produces).

Motion channels — head, target, scene

Each is a list of 6-degree-of-freedom segments. head drives the canals and otoliths; target is the foveal target in 3-D space (its z is the viewing distance); scene is the full-field visual background (drives OKR/OKN). Angles are in degrees, positions in metres (x = right, y = up, z = forward/depth). A field left unset carries over from the previous segment.

Patient

Overrides of model parameters from their healthy defaults — canal gains (vestibular loss), velocity-storage and neural-integrator time constants, saccade / pursuit / vergence gains, cerebellar gains, cranial-nerve / nucleus / MLF lesions, the plant, and more. See the parameters page for the complete list with units, ranges, and anatomical meaning.

Plot

Which signal panels to show. Core readouts (eye position, eye velocity, vergence) lead; for nystagmus add slow-phase velocity; then the driving stimulus and the internal-mechanism panel the test probes. The underlying signals are documented on the states & activations page.

Reading the eye-position panel. Both eyes (L, R) are always drawn so you can see binocular behaviour — cover test, phoria/tropia, nerve palsy, skew. Because each eye must point at a slightly different angle to fixate a near target (that difference is the convergence), the horizontal axis is zero-referenced so the traces stay readable at any distance, and what 0 means is stated in the axis label:

  • 0 = on target @ distance — a target at a fixed distance: both eyes sit on the target line when fixating, and any misalignment makes them split apart (the cover test deviation, a palsy, etc.).
  • 0 = resting position — no target (e.g. in the dark): both eyes read ~0 at rest, so drift and nystagmus are read around zero.
  • 0 = straight ahead — a target whose distance changes: absolute angles, so the convergence/divergence movements stay visible (the vergence panel shows actual vs required).

This is a display calibration only — the downloaded data and the underlying simulation keep the raw, absolute eye angles.

Comparison

Two or more scenarios can be run and overlaid on the same panels to contrast conditions — e.g. healthy vs lesioned, or with vs without a manipulation.

ViSiOMlab is a research and educational tool. It is a model of oculomotor physiology, not a medical device, and is not intended for clinical diagnosis or treatment.
